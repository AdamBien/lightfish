LightFish - Escalation - Message Help

Escalation messages can be used to inform the reason for the escalation.

Message types

There are two types of escalation messages. There is a basic message, and a rich message. The basic message is plain text and will be used when rich content cannot be displayed. The rich message uses a MarkDown processor, specifically PegDown, to create an HTML message.

Variable Replacement

In your message you may want to reference specific values from the the current data; to do this you add text like '\${propertyName}', where the property name is the property you want to access. You have access to all the properties of the current snapshot that caused the escalation. Here is a list of the property names:

  • monitoringTime
  • usedHeapSize
  • threadCount
  • peakThreadCount
  • totalErrors
  • currentThreadBusy
  • committedTX
  • rolledBackTX
  • queuedConnections
  • activeSessions
  • expiredSessions
